## ğŸ“ˆ Usage

### Generate New Reports

```bash
# Display in console only
npm run analyze

# Save to file and display in console
npm run analyze:save

# Save as JSON (quiet mode)
npm run analyze:json

# Save to file only (no console output)
npm run analyze:quiet
```

### Compare Reports

Use the `index.json` file to track all reports chronologically. The reports are sorted by timestamp (newest first) and include key metrics for quick comparison.

### Manual Comparison

You can compare any two reports by:
1. Opening two report files side by side
2. Using `diff` command: `diff report1.txt report2.txt`
3. For JSON reports, use comparison tools or scripts

## ğŸ“‹ Report Contents

### Overview Section
- Total Files: Number of source files analyzed
- Total Lines: Total lines across all files
- Code Lines: Actual code lines (excluding comments and blanks)
- Comment Lines: Lines containing comments
- Blank Lines: Empty or whitespace-only lines

### File Type Breakdown
Statistics grouped by file extension showing:
- Number of files per type
- Total lines per type
- Code lines per type

### Directory Breakdown
Statistics grouped by directory showing:
- Files per directory
- Code lines per directory

### Summary Statistics
- Average lines per file
- Code density percentage
- Total file types and directories analyzed
